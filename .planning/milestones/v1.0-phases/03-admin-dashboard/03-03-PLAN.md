<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → drivers server actions | Driver mutations are publicly-addressable POST endpoints reachable without page navigation |
| client form → server | Untrusted driver name/phone input crosses into the DB |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-11 | Elevation of Privilege | unauthenticated direct call to addDriver/updateDriver/setDriverActive/deleteDriver | mitigate | `requireSession()` (verifies `session.isLoggedIn` via iron-session) is the FIRST statement of every mutation action; throws if absent (Pitfall 5 — middleware does not protect server actions) |
| T-03-12 | Tampering | malformed / oversized driver input | mitigate | zod `driverSchema` with E.164 regex + length caps (name ≤120, phone ≤20) via `safeParse`; reject with error message, never persist invalid data |
| T-03-13 | Tampering | CSRF on driver mutations | mitigate | Next.js 16 Server Actions enforce same-origin via the framework's built-in Action origin check (RESEARCH A1); actions are not exposed as open REST routes |
| T-03-14 | Repudiation | accidental irreversible driver delete | mitigate | Hard delete gated behind DeleteConfirmDialog (explicit confirm); deactivate (reversible) is the default non-destructive path (D-10, UI-08) |
</threat_model>
