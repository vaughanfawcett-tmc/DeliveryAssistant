## Spacing Scale

Declared values (multiples of 4). Reuse Phase 2 Tailwind utilities directly — no new tokens.

| Token | Tailwind class | Value | Usage |
|-------|---------------|-------|-------|
| xs | gap-1 / p-1 | 4px | Icon gaps, badge padding, tight inline spacing |
| sm | gap-2 / p-2 | 8px | Button internal padding (vertical), chip padding |
| md | gap-4 / p-4 | 16px | Default card padding, form field vertical gap, table cell padding |
| lg | gap-6 / p-6 | 24px | Card-to-card gap, section internal padding |
| xl | gap-8 / p-8 | 32px | Sidebar width inner padding, page content horizontal padding (desktop) |
| 2xl | gap-12 / py-12 | 48px | Page top/bottom breathing room |
| 3xl | gap-16 / py-16 | 64px | Login page vertical centering offset |

Exceptions:
- Touch targets (nav links, table row action buttons): minimum 44px height regardless of content — add `min-h-[44px]` to achieve this.
- Sidebar nav width: 240px fixed on desktop, collapses to full-width drawer on mobile.
- Metrics cards: 2-column grid on ≥640px, 1-column on <640px.

## Typography

Carries forward Phase 2 conventions exactly [codebase: StatusHeader.tsx, LookupForm.tsx, EventHistory.tsx].

| Role | Size | Tailwind | Weight | Tailwind | Line Height | Usage |
|------|------|----------|--------|----------|-------------|-------|
| Body | 14px | text-sm | 400 regular | font-normal | 1.5 | Table cell content, transcript body text, helper text |
| Label | 14px | text-sm | 600 semibold | font-semibold | 1.4 | Form labels, column headers, nav link text, filter labels |
| Heading | 20px | text-xl | 600 semibold | font-semibold | 1.25 | Page headings (Metrics, Call History, Drivers), modal titles |
| Display | 28px | text-2xl | 600 semibold | font-semibold | 1.2 | Login page heading only |

Secondary type roles:
- Metric value (large stat): 32px / text-3xl / font-semibold — reserved for the 4 call metric numbers on the Metrics page.
- Metric label (under stat): 14px / text-sm / font-semibold / text-zinc-500 — "Today", "7 days", "30 days" period labels; text-zinc-500 provides visual distinction from body text at the same size.
- Monospace (tracking refs, phone numbers, timestamps): font-mono (Geist Mono) / text-sm — call history table ref column, driver phone column, transcript timestamps.
- Caption: 14px / text-sm / font-normal / text-zinc-400 — table pagination counts, recording duration, "last updated" notes; text-zinc-400 provides visual distinction from body text at the same size.

Maximum declared sizes: 4 (14, 20, 28, 32). Maximum declared weights: 2 (regular 400 + semibold 600).

## Color

Extends Phase 2 token set exactly. New semantic additions for admin states are Tailwind utilities — no new CSS vars.

| Role | Hex / Token | Tailwind | Usage |
|------|-------------|----------|-------|
| Dominant (60%) | #ffffff light / #0a0a0a dark | bg-background | Page background, sidebar background, login page |
| Secondary (30%) | zinc-50 (#fafafa) | bg-zinc-50 | Cards, table rows (even), input backgrounds, detail panels |
| Accent (10%) | #2563eb | bg-accent / text-accent | [see reserved list below] |
| Destructive | red-600 (#dc2626) | text-red-600 / border-red-500 | Delete driver button + confirm dialog destructive action only |
| Neutral border | zinc-200 | border-zinc-200 | Card borders, table row dividers, input borders |
| Muted text | zinc-500 | text-zinc-500 | Helper text, empty-state body, table secondary columns |
| Success / contained | green-600 (#16a34a) | text-green-600 | Outcome badge "resolved" |
| Warning / missed | amber-600 (#d97706) | text-amber-600 | Outcome badge "missed" |
| Escalated | zinc-500 | text-zinc-500 | Outcome badge "escalated" / "failed" (neutral, not alarming) |

Accent (#2563eb) reserved for:
1. Primary CTA buttons (Login, Save driver, Add driver)
2. Active nav link indicator (left border or text color)
3. Focus ring on all interactive inputs (focus:ring-accent, already in Phase 2)
4. Metric trend up-arrow / positive indicator icon
5. Filter "active" chip/pill background
6. Links within transcript and call detail view

Accent is NOT used for:
- Table row hover (use zinc-50/zinc-100 hover states)
- Outcome badges (use semantic green/amber/zinc)
- Sidebar background (use bg-background)

## Component Inventory

All components are hand-authored to match Phase 2 conventions. No shadcn or third-party UI blocks.

### Layout Shell (new — admin only)

**AdminShell**
- Left sidebar: 240px fixed on ≥1024px, off-canvas drawer on <1024px
- Sidebar contains: Derby Aggs wordmark (text, same style as portal h1), nav links (Metrics, Call History, Drivers), logout button at bottom
- Main content area: flex-1, overflow-y-auto, horizontal padding px-4 (mobile) / px-8 (desktop)
- Top bar on mobile: hamburger button (44×44px touch target) + page title + logout icon

**Nav link states:**
- Default: text-zinc-600, no background
- Hover: text-zinc-900, bg-zinc-50
- Active (current route): text-accent, border-l-2 border-accent, font-semibold

### Login Page (ADMIN-01)

**LoginForm** (client component)
- Centered card: max-w-sm, border border-zinc-200, rounded-xl, p-8, bg-background
- "Derby Aggs — Staff login" heading: text-xl font-semibold text-zinc-900
- Single password field: type="password", same input style as Phase 2 LookupForm (rounded-lg, border-zinc-300, px-4 py-3, focus:ring-2 focus:ring-accent)
- Submit button: full-width, rounded-full, bg-accent text-white h-12 — same as portal CTA
- Error state: red-600 text below the field, role="alert", icon-free (text only matches Phase 2 pattern)
- No username field — shared password only (D-04)

### Metrics Page (ADMIN-02)

**PeriodTabs**
- 3 tab buttons: "Today" / "7 days" / "30 days"
- Active tab: text-accent, border-b-2 border-accent
- Inactive: text-zinc-500, hover:text-zinc-700
- Tab strip: border-b border-zinc-200, px-0

**MetricCard** (×4)
- White card, border border-zinc-200, rounded-xl, p-6
- Metric value: text-3xl font-semibold text-zinc-900
- Metric name: text-sm font-semibold text-zinc-500 mt-1
- Cards: "Total received", "Answered", "Missed", "Success rate"
- Grid: 2-column on ≥640px, 1-column on <640px, gap-4

**Empty state (no calls yet):**
- Same card grid layout, all values show "—"
- Below grid: single centered paragraph "No calls recorded yet. Data will appear here once the voice agent is live." — text-sm text-zinc-500

### Call History Page (ADMIN-03)

**CallFilters** (client component)
- Date range: two date inputs (from/to), text-sm, border-zinc-200, rounded-lg, px-3 py-2
- Outcome select: "All outcomes" / "Resolved" / "Missed" / "Escalated" / "Failed"
- Search input: placeholder "Search tracking reference…", debounced 300ms
- Filter row: flex-wrap gap-3 on desktop, stacked on mobile
- Active filter chip: bg-accent/10 text-accent text-sm font-semibold rounded-full px-3 py-1 with × dismiss

**CallHistoryTable**
- Columns: Date/time | Duration | Tracking ref | Outcome | Caller (masked) | Action
- Date/time: font-mono text-sm text-zinc-600
- Duration: text-sm text-zinc-600 (e.g. "1m 42s")
- Tracking ref: font-mono text-sm text-zinc-900
- Outcome badge: rounded-full px-2 py-0.5 text-sm font-semibold — green-600/green-100 (resolved), amber-600/amber-100 (missed), zinc-600/zinc-100 (escalated/failed)
- Caller: font-mono text-sm text-zinc-500 (masked: last 4 digits, e.g. "••• ••• 1234")
- Action: "View" link — text-accent text-sm underline
- Row hover: bg-zinc-50
- Mobile breakpoint (<768px): table collapses to stacked cards — each row becomes a card (border-zinc-200, rounded-lg, p-4, mb-2) with label/value pairs

**Empty state (no matching calls):**
- Heading: "No calls found" text-base font-semibold text-zinc-900
- Body: "Try adjusting your filters or date range." text-sm text-zinc-500
- If table is empty AND no filters active: "No calls recorded yet. Voice agent data will appear here after Phase 4." text-sm text-zinc-500

**Pagination:**
- "Showing 1–25 of 47 calls" caption text-sm text-zinc-400
- Previous / Next buttons: text-sm text-zinc-600, disabled:opacity-40
- Page size: 25 rows default

### Call Detail Panel (ADMIN-03 / ADMIN-04 / ADMIN-05 / ADMIN-07)

**CallDetail** — full-page route (not a modal, given transcript length)
- Back link: "← Call history" text-accent text-sm — top-left
- Metadata section: 2-column grid (date/time, duration, tracking ref, outcome, caller) — label text-sm text-zinc-500, value text-sm font-semibold text-zinc-900

**TranscriptView** (ADMIN-04)
- Scrollable panel, max-height 60vh on desktop, full height on mobile
- Each turn: speaker label (text-sm font-semibold text-zinc-500, "Agent" or "Customer") + timestamp (font-mono text-sm text-zinc-400) + message text (text-sm text-zinc-900 leading-relaxed)
- Agent turns: left-aligned, no background
- Customer turns: left-aligned, bg-zinc-50 rounded-lg px-3 py-2
- Empty state: "Transcript not available for this call." text-sm text-zinc-500

**RecordingPlayer** (ADMIN-05)
- Native HTML `<audio controls>` element — no custom player (simplest implementation, browser-native, accessible)
- Wrapper: border border-zinc-200 rounded-lg p-4 bg-zinc-50
- Title: "Call recording" text-sm font-semibold text-zinc-700 mb-2
- Unavailable state: "Recording not yet available. Recordings will be stored for 30 days after the voice agent goes live." text-sm text-zinc-500

**DriverCallSubLog** (ADMIN-07)
- Collapsible section below transcript: "Outbound driver calls (N)" heading, expands on click
- Sub-table: Driver name | Phone | Duration | Outcome — same font/badge conventions as CallHistoryTable
- Empty state: "No outbound driver calls for this call." text-sm text-zinc-500

### Drivers Page (ADMIN-06)

**DriverList**
- Table on desktop: Name | Phone | Status | Actions
- Status badge: "Active" (green-600/green-100) or "Inactive" (zinc-500/zinc-100)
- Phone: font-mono text-sm
- Actions: "Edit" (text-accent text-sm) + "Deactivate"/"Activate" (text-zinc-600 text-sm) + "Delete" (text-red-600 text-sm)
- Mobile: stacked cards (same collapse pattern as CallHistoryTable)
- "Add driver" button: top-right on desktop, sticky bottom on mobile — bg-accent text-white rounded-full px-5 py-2 text-sm font-semibold

**Empty state (no drivers):**
- Heading: "No drivers yet" text-base font-semibold text-zinc-900
- Body: "Add your first driver to enable outbound call escalation." text-sm text-zinc-500
- CTA: same "Add driver" button style

**DriverModal** (client component — add/edit) (D-10)
- Dialog: role="dialog", centered overlay, max-w-sm, rounded-xl, shadow-lg, p-6, bg-background
- Backdrop: fixed inset-0 bg-black/40
- Fields: "Full name" (text input) + "Phone number" (text input, E.164 placeholder "+44 7911 123456")
- Validation inline: red-600 text-sm below field, visible only after blur
- E.164 validation: client-side regex + server-side zod refinement
- Save button: full-width bg-accent text-white rounded-full h-11 font-semibold
- Cancel: text-zinc-500 text-sm underline below button — label "Discard changes"
- Modal title: "Add driver" or "Edit driver" — text-xl font-semibold

**DeleteConfirmDialog** (client component) (D-10)
- Separate dialog, triggered by "Delete" action
- Title: "Delete [driver name]?" — text-lg font-semibold text-zinc-900
- Body: "This will permanently remove them from the system. This cannot be undone." text-sm text-zinc-600
- Buttons: "Keep driver" (border border-zinc-300 rounded-full px-5 py-2 text-sm text-zinc-700) + "Delete driver" (bg-red-600 text-white rounded-full px-5 py-2 text-sm font-semibold)
- Deactivate is reversible — no confirmation dialog; immediate with inline status toggle and toast feedback

**Toast / feedback**
- Single-line toast: fixed bottom-4 left-1/2 -translate-x-1/2, bg-zinc-900 text-white text-sm rounded-full px-4 py-2, auto-dismisses after 3s
- Messages: "Driver saved", "Driver deleted", "Driver deactivated", "Driver activated"
- Error toast: bg-red-600 text-white

## State and Loading Patterns

**Skeleton loaders** — used on initial server-component data fetch for:
- MetricCard ×4: shimmer rectangles matching card dimensions (animate-pulse bg-zinc-200)
- CallHistoryTable: 5 skeleton rows of height 48px, shimmer
- DriverList: 3 skeleton rows

**Suspense boundaries:** each major data section wrapped in React Suspense with skeleton fallback.

**Loading spinners** — not used for full-page loads; reserved only for inline form submission (Login button, Save driver button) — show spinner icon inside button while pending.

**Error boundaries:** each major route has an error.tsx that shows:
- Heading: "Something went wrong" text-xl font-semibold text-zinc-900
- Body: "We couldn't load this page. Please try refreshing." text-sm text-zinc-600
- Refresh link: text-accent text-sm underline

## Accessibility Requirements

- All form inputs have associated `<label>` elements (htmlFor/id pair).
- Dialogs use role="dialog", aria-modal="true", aria-labelledby pointing to dialog title, focus trapped on open, returns focus on close.
- Outcome badges have aria-label for screen readers (not just colour coding).
- Metric cards use `<h2>` for metric name (within the section role context).
- Audio player is native `<audio controls>` — browser-native accessibility.
- Transcript speaker labels use `<span aria-hidden="true">` display prefix + logical reading order.
- Min tap target 44×44px on all interactive elements (nav, buttons, table actions).

## Design Decisions Log

Decisions made by this researcher with rationale, annotated by source.

| # | Decision | Rationale | Source |
|---|----------|-----------|--------|
| UI-01 | No shadcn — hand-authored Tailwind v4 | Phase 2 established hand-authored pattern; introducing shadcn mid-project would cause style inconsistency | [codebase] |
| UI-02 | Persistent sidebar at lg breakpoint (1024px), off-canvas drawer below | Standard admin dashboard pattern; lg is the first breakpoint where 240px sidebar + content doesn't feel cramped | [assumption — standard pattern] |
| UI-03 | MetricCard grid: 2-col at sm, 1-col below | 4 cards at 2-col fits 640px+ comfortably; 1-col for phones is clean | [assumption] |
| UI-04 | Call detail as full route, not modal | Transcripts can be long; modal scroll with backdrop is poor UX; a dedicated URL is bookmarkable/sharable | [assumption — standard pattern] |
| UI-05 | Native `<audio controls>` for recording | No custom player needed in v1; native is accessible, zero dependencies, matches project's no-extra-lib philosophy | [upstream D-03] |
| UI-06 | Table-to-card collapse at <768px | md breakpoint is the natural table-to-card crossover for this column density | [assumption] |
| UI-07 | Toast for driver CRUD feedback | Modals already used for add/edit; non-blocking toast is correct for success feedback | [assumption — standard pattern] |
| UI-08 | Destructive red-600 only for Delete driver, not for Deactivate | Deactivate is reversible (D-10); red should signal irreversible actions only | [upstream D-10] |
| UI-09 | Outcome badges use semantic colours (green/amber/zinc) not accent | Accent is reserved for brand/interactive; outcome is status information — semantic coding | [assumption] |
| UI-10 | Monospace font for tracking refs, phone numbers, timestamps | Geist Mono already loaded; tabular alignment aids scanning in tables | [codebase — font already present] |
| UI-11 | 25-row pagination default | Balances page load and scrolling; short enough to paginate without virtual scroll complexity | [assumption] |
| UI-12 | Period tabs (Today/7d/30d) rather than date-picker on Metrics | ADMIN-02 specifies exactly these three windows; a date picker is overkill for a fixed-window summary | [upstream ADMIN-02] |
| UI-13 | 12px caption/label tier removed; text-zinc-400/500 provides visual hierarchy at 14px | Two weights (400/600) + colour differentiation is sufficient for distinction; eliminating the 12px tier keeps the type scale to 4 declared sizes and avoids the 5-size breach flagged by checker | [checker revision 2026-06-12] |
| UI-14 | 500/medium weight removed; all label-weight elements promoted to 600/semibold | Three declared weights exceeded the 2-weight maximum; merging medium into semibold is unambiguous and consistent with Phase 2 semibold-for-emphasis convention | [checker revision 2026-06-12] |
