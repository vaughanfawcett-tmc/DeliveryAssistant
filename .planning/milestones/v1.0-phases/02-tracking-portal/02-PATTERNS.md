### `src/app/actions/lookup.ts` (server action — POST lookup, PORT-01)

**Analog:** `src/lib/repositories/lookup-log.ts` — same factory + lazy-init singleton export pattern (lines 53–149)

**Server action structure to copy:**
```typescript
'use server';

import { lookupConsignment } from '@/lib/tracking/service';
import type { TrackingResult } from '@/types/tracking';

export async function lookup(formData: FormData): Promise<TrackingResult> {
  const trackingRef = (formData.get('trackingRef') as string | null)?.trim() ?? '';
  const postcode    = (formData.get('postcode')    as string | null)?.trim() ?? '';

  // Minimal guard — blank inputs return not_found rather than throwing
  if (!trackingRef || !postcode) {
    return { ok: false, reason: 'not_found' };
  }

  return lookupConsignment({ trackingRef, postcode });
}
```

**Error handling pattern** — mirrors `lookup-log.ts` lines 70–73: never throw to the caller; map errors to a typed discriminated-union result. The `TrackingResult` union carries all error states — no try/catch needed in the action when `lookupConsignment` already handles API errors internally.

**Import alias:** Use `@/lib/...` (tsconfig path alias already present from Phase 1 scaffold).

### `src/lib/share/token.ts` (utility — signed share link, PORT-08)

**Analog:** `src/lib/tracking/postcode.ts` — same pure-utility module shape (no class, named function exports, no side effects)

**Postcode utility pattern to mirror** (from `src/lib/tracking/postcode.ts` — inferred from service.ts imports):
```typescript
// Pure functions, no default export, named exports only
export function normalisePostcode(raw: string): string { ... }
export function postcodesMatch(a: string, b: string): boolean { ... }
```

**Token utility shape to follow:**
```typescript
import { createHmac } from 'crypto';
import { env } from '@/lib/env';

const SEPARATOR = '.';
const TTL_SECONDS = 60 * 60 * 24; // 24 h default

/** Produce a signed share token encoding the consignment number + expiry. */
export function createShareToken(consignmentNumber: string): string {
  const exp = Math.floor(Date.now() / 1000) + TTL_SECONDS;
  const payload = Buffer.from(JSON.stringify({ c: consignmentNumber, exp })).toString('base64url');
  const sig = sign(payload);
  return `${payload}${SEPARATOR}${sig}`;
}

/** Verify a share token. Returns the consignment number or null if invalid/expired. */
export function verifyShareToken(token: string): string | null { ... }

function sign(payload: string): string {
  return createHmac('sha256', env.SHARE_TOKEN_SECRET).update(payload).digest('base64url');
}
```

**Rule:** HMAC-SHA256 over `base64url(JSON({c, exp}))`. The signed token encodes only the consignment number — never the postcode. Verification returns `null` on bad signature or expired token so the route handler can return 404 safely.

**New env field required** — add `SHARE_TOKEN_SECRET: z.string().min(32)` to `envSchema` alongside `CONTACT_PHONE`.

### `src/app/track/[token]/page.tsx` (page — signed share view, PORT-08)

**Analog:** `src/app/page.tsx` scaffold — same Server Component shape

**Route handler pattern:**
```tsx
// No "use client" — Server Component
import { verifyShareToken } from '@/lib/share/token';
import { lookupConsignment } from '@/lib/tracking/service';
import { notFound } from 'next/navigation';

interface Props { params: { token: string } }

export default async function SharePage({ params }: Props) {
  const consignmentNumber = verifyShareToken(params.token);
  if (!consignmentNumber) notFound();

  // Re-fetch — postcode is not in the token; use a sentinel that bypasses
  // the postcode gate in a dedicated "share lookup" path, OR store the
  // consignment number and look up without postcode re-verification
  // (planner to resolve the exact bypass mechanism)
  const result = await lookupConsignment({ trackingRef: consignmentNumber, postcode: '__share__' });
  if (!result.ok) notFound();

  return <TrackingResult consignment={result.consignment} readOnly />;
}
```

**Print stylesheet pattern (D-12):** Add `<link rel="stylesheet" href="/print.css" media="print" />` in the layout, or use Tailwind's `print:` variant. No separate page component needed — `print:hidden` on share/form controls.

### `src/components/MilestoneStepper.tsx` (component — PORT-02, D-07 / discretion)

```typescript
  'booked', 'at_hub', 'in_transit', 'out_for_delivery', 'delivered',
```

**Component pattern:**
```tsx
import { MILESTONE_ORDER, type MilestoneStage } from '@/types/tracking';

const LABELS: Record<MilestoneStage, string> = {
  booked:           'Booked',
  at_hub:           'At hub',
  in_transit:       'On its way',
  out_for_delivery: 'Out for delivery',
  delivered:        'Delivered',
};

interface Props { currentStage: MilestoneStage }

export function MilestoneStepper({ currentStage }: Props) {
  const activeIdx = MILESTONE_ORDER.indexOf(currentStage);
  return (
    <ol className="flex items-center justify-between w-full gap-1" aria-label="Delivery progress">
      {MILESTONE_ORDER.map((stage, idx) => {
        const done    = idx <= activeIdx;
        const active  = idx === activeIdx;
        return (
          <li key={stage} className="flex flex-col items-center flex-1 text-center">
            <div className={`w-6 h-6 rounded-full flex items-center justify-center text-xs font-bold
              ${done ? 'bg-accent text-white' : 'bg-zinc-200 text-zinc-400'}`}>
              {done ? '✓' : idx + 1}
            </div>
            <span className={`mt-1 text-[10px] leading-tight
              ${active ? 'text-accent font-semibold' : done ? 'text-zinc-700' : 'text-zinc-400'}`}>
              {LABELS[stage]}
            </span>
          </li>
        );
      })}
    </ol>
  );
}
```

**Design rule (D-07 / discretion):** Horizontal 5-step stepper, active and completed steps in `text-accent` / `bg-accent`. Steps before active are filled; steps after are grey. Fits 375px because each step is `flex-1` with `text-[10px]` label.

### `src/components/ErrorState.tsx` (component — PORT-05, D-11)

**Analog:** `src/types/tracking.ts` — `LookupFailureReason` union is the data contract

**LookupFailureReason union** (lines 14–18):
```typescript
export type LookupFailureReason =
  | 'not_found'
  | 'postcode_mismatch'
  | 'multiple_matches'
  | 'api_error';
```

**Component pattern — one distinct message per reason (D-11):**
```tsx
import { env } from '@/lib/env';
import type { LookupFailureReason } from '@/types/tracking';
import type { NexusConsignment } from '@/types/consignment'; // for multiple_matches candidates

interface Candidate { consignmentNumber: string; delAddressTown: string | null; plainStatus: string }

interface Props {
  reason: LookupFailureReason;
  candidates?: Candidate[]; // only present when reason === 'multiple_matches'
  onSelect?: (consignmentNumber: string) => void;
}

const MESSAGES: Record<LookupFailureReason, { heading: string; body: string }> = {
  not_found: {
    heading: "We couldn't find that delivery",
    body: "Check your tracking number and make sure you're using the delivery postcode.",
  },
  postcode_mismatch: {
    heading: "Postcode doesn't match",
    body: "The postcode you entered doesn't match the delivery address for that tracking number.",
  },
  multiple_matches: {
    heading: "Multiple deliveries found",
    body: "We found more than one delivery. Please choose yours below.",
  },
  api_error: {
    heading: "Service unavailable",
    body: "We can't reach the delivery system right now. Please try again in a moment.",
  },
};

export function ErrorState({ reason, candidates, onSelect }: Props) {
  const { heading, body } = MESSAGES[reason];
  const showCallUs  = reason === 'not_found' || reason === 'api_error';

  return (
    <div role="alert" className="flex flex-col gap-4 w-full max-w-md">
      <h2 className="text-lg font-semibold text-zinc-900">{heading}</h2>
      <p className="text-zinc-600">{body}</p>

      {/* Multiple-match chooser — D-10 */}
      {reason === 'multiple_matches' && candidates && (
        <ul className="flex flex-col gap-2">
          {candidates.map(c => (
            <li key={c.consignmentNumber}>
              <button onClick={() => onSelect?.(c.consignmentNumber)}
                className="w-full text-left px-4 py-3 rounded-lg border border-zinc-200
                           hover:border-accent hover:bg-accent/5 transition-colors">
                <span className="font-mono text-sm">{c.consignmentNumber}</span>
                {c.delAddressTown && <span className="text-zinc-500 ml-2">— {c.delAddressTown}</span>}
                <span className="block text-xs text-zinc-500 mt-0.5">{c.plainStatus}</span>
              </button>
            </li>
          ))}
        </ul>
      )}

      {/* Dead-end prevention — D-11 */}
      {showCallUs && (
        <p className="text-sm text-zinc-600">
          Need help?{' '}
          <a href={`tel:${env.CONTACT_PHONE}`} className="text-accent underline font-medium">
            Call us on {env.CONTACT_PHONE}
          </a>
        </p>
      )}

      {/* Retry link always present */}
      <a href="/" className="text-sm text-accent underline">Try again</a>
    </div>
  );
}
```

**Rule:** Every branch of `LookupFailureReason` has an explicit entry in `MESSAGES` — TypeScript will fail to compile if a new reason is added without updating this map. `CONTACT_PHONE` comes from `env`, not hardcoded (D-11).

## Shared Patterns

### Server Component default (Next.js App Router)
**Source:** `src/app/page.tsx` scaffold and `src/app/layout.tsx` (lines 20–33)
**Apply to:** All `src/app/**/*.tsx` files

No `"use client"` directive unless the component needs browser APIs or event handlers. Forms use `action={serverAction}` directly — no `useFormState`/`useFormStatus` unless pending-state feedback is needed in Phase 2 (treat as Claude's discretion).

### TypeScript import style
**Source:** `src/lib/tracking/service.ts` (lines 19–23), `src/lib/env.ts` (lines 1–2)
**Apply to:** All new files

```typescript
import type { Foo } from '...';  // type-only imports separated from value imports
import { bar } from '...';       // value imports on separate lines
```

Path alias `@/` maps to `src/` — use it for all cross-directory imports (matches Phase 1 tsconfig convention).

### Tailwind class ordering
**Source:** `src/app/page.tsx` scaffold (lines 5–63)
**Apply to:** All component files

Layout (`flex`/`grid`/`w-`/`h-`) → spacing (`p-`/`m-`/`gap-`) → typography (`text-`/`font-`/`leading-`) → colour (`bg-`/`text-`/`border-`) → state (`hover:`/`focus:`/`active:`) — matches the scaffold's class ordering convention.

### Named function exports (no default exports for utilities/components)
**Source:** `src/lib/tracking/service.ts` (lines 50, 146), `src/lib/tracking/status-map.ts` (line 75)
**Apply to:** `src/lib/share/token.ts`, all component files

```typescript
// Preferred — named exports, no default
export function createShareToken(...) { ... }
export function verifyShareToken(...) { ... }
```

Exception: Next.js pages/layouts require `export default function` — use default only there.

### Factory + lazy singleton export pattern
**Source:** `src/lib/repositories/lookup-log.ts` (lines 53–149), `src/lib/tracking/service.ts` (lines 50–152)
**Apply to:** Any new `src/lib/` module that needs DI for testing

```typescript
// 1. Export a factory function for tests to inject deps
export function createFoo(deps: FooDeps) { ... }

// 2. Export a singleton-bound function for production callers
let _default: ReturnType<typeof createFoo> | undefined;
export async function fooAction(...) {
  if (!_default) { _default = createFoo(realDeps); }
  return _default.fooAction(...);
}
```

### Zod env extension
**Source:** `src/lib/env.ts` (lines 3–34)
**Apply to:** `src/lib/env.ts` (modify)

Add new env fields inside the existing `envSchema` `.object({})` block — one field per line, same zod chain style as existing fields. Never access `process.env` directly outside `env.ts`.

### Test file structure
**Source:** `src/lib/tracking/service.test.ts` (lines 1–146)
**Apply to:** `src/lib/share/token.test.ts`, any action/utility tests

```typescript
import { describe, it, expect, vi, beforeEach } from 'vitest';
// ... imports

describe('module name (via factory or direct)', () => {
  // setup in beforeEach
  it('expected outcome: describe the condition and result', async () => {
```
