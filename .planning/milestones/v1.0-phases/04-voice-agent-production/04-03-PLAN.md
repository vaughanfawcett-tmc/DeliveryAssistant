## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| app → ElevenLabs/Twilio REST | Outbound calls cost money + can loop; auth secrets cross here |
| driver phone number → outbound call | E.164 number is PII and a toll-fraud target |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-09 | Elevation/abuse | placeOutboundCall | mitigate | Adapter exposes only place/end/transfer/dtmf; it does not accept arbitrary URLs, removing SSRF surface — the destination is always a managed driver E.164 (resolved in Plan 05 via getDriverById) |
| T-04-10 | Information disclosure | auth headers | mitigate | Bearer/Basic creds are injected config, never logged; real adapter reads them lazily via factory so they never land in test snapshots |
| T-04-11 | Denial of service / toll fraud | outbound calls | transfer | Hard duration/retry limits are enforced by the Plan 05 escalation machine + DRIVER_CALL_MAX_* env; the adapter itself is a thin transport |
| T-04-12 | Spoofing | Twilio webhook origin | mitigate | Handled in Plan 04 (X-Twilio-Signature HMAC verify); this plan only builds the outbound transport |
</threat_model>
