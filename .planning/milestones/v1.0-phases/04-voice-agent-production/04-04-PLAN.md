<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ElevenLabs/Twilio → /api/voice/* | PUBLIC unauthenticated internet endpoints; primary attack surface |
| webhook body → lookupConsignment | Untrusted JSON drives a delivery lookup |
| webhook body → calls table | Untrusted JSON persists transcript/PII |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-13 | Spoofing | all /api/voice/* handlers | mitigate | Constant-time HMAC verify (per provider) runs FIRST; unsigned/invalid -> 401 before any parse/lookup/DB write (test-enforced 401-first) |
| T-04-14 | Tampering | request bodies | mitigate | zod-validate every payload; invalid -> 400; signature covers raw body so tamper invalidates it |
| T-04-15 | Information disclosure | lookup_consignment response | mitigate | Response carries ONLY TrackingResult fields; postcode gate already enforced inside lookupConsignment; nothing leaks for a mismatch |
| T-04-16 | Repudiation | call_ended | mitigate | Persists transcript + outcome + disconnection_reason so every call has an auditable record (DRIV-04 customer leg) |
| T-04-17 | Information disclosure | from_number in calls | accept | Stored server-side on an RLS table; masked by maskPhone at the dashboard read boundary (existing) |
| T-04-18 | Replay | webhook signature | accept | Read-only/idempotent-ish persistence keyed on unique platform_call_id; provider replay value is low — timestamp-nonce hardening deferred to Plan 06 runbook (provider-level) |
</threat_model>
