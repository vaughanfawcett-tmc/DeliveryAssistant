<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser form -> server action | Untrusted trackingRef/postcode cross into the server here |
| share URL (token) -> server route | Untrusted token string crosses at /track/[token] |
| result -> share URL | The minted share URL is the only place a consignment identity leaves the server to the customer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-12 | Information Disclosure | lookup form transport | mitigate | Form uses `action={serverAction}` (POST/RSC action), never method=GET; trackingRef + postcode never serialise into the URL or browser history (PORT-01, D-05); acceptance grep confirms no GET method |
| T-02-13 | Information Disclosure | share URL contents | mitigate | /track/[token] URL contains only the HMAC token from createShareToken (encodes `{c,exp}`); the share page reads no postcode and renders read-only; file asserted to contain no `postcode` reference (PORT-08, D-12) |
| T-02-14 | Spoofing / Tampering | /track/[token] forged or expired token | mitigate | verifyShareToken (HMAC + expiry, Plan 02) gates the route; null → `notFound()` (404) before any Nexus call; unit-tested valid→resolve / invalid→null |
| T-02-15 | Denial of Service | oversized form input | mitigate | Server action caps trackingRef at 30 and postcode at 20 chars before calling the service (PITFALLS.md input-length limit); over-length input returns not_found |
| T-02-16 | Spoofing | share-link minting | accept | makeShareUrl runs server-side and only mints a token for a consignment number the session already resolved through the postcode gate; no client-supplied consignment number is signed without that prior gated lookup |
</threat_model>
