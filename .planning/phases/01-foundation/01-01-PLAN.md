Mapped result types. Create in src/types/tracking.ts:

```typescript
export type MilestoneStage =
  | 'booked'
  | 'at_hub'
  | 'in_transit'
  | 'out_for_delivery'
  | 'delivered';

export const MILESTONE_ORDER: MilestoneStage[] = [
  'booked', 'at_hub', 'in_transit', 'out_for_delivery', 'delivered',
];

export type LookupFailureReason =
  | 'not_found'
  | 'postcode_mismatch'
  | 'multiple_matches'
  | 'api_error';

export interface MappedConsignment {
  consignmentNumber: string;
  plainStatus: string;        // plain-language status label (API-04)
  description: string;        // plain-language description (API-04)
  currentStage: MilestoneStage;
  estimatedDelDate: string | null;
  startWindow: string | null;
  endWindow: string | null;
  routeDetails: NexusRouteDetail[];
}

export type TrackingResult =
  | { ok: true; consignment: MappedConsignment }
  | { ok: false; reason: LookupFailureReason };
```

- `PALLEX_USERNAME`: `z.string().min(1)` (credentials live in env secrets only — PITFALLS.md)
- `PALLEX_PASSWORD`: `z.string().min(1)`
- `SUPABASE_URL`: `z.string().url()`
- `SUPABASE_SERVICE_ROLE_KEY`: `z.string().min(1)`
- `UPSTASH_REDIS_REST_URL`: `z.string().url()`
- `UPSTASH_REDIS_REST_TOKEN`: `z.string().min(1)`

When PALLEX_MOCK is true, PALLEX_USERNAME/PASSWORD/BASE_URL may be placeholder values, so make them `.default('mock')` ONLY when mock is true is hard in a flat schema — instead use `.optional()` for PALLEX_USERNAME and PALLEX_PASSWORD with a refine: if PALLEX_MOCK is false, both must be non-empty (`.refine(...)` at the object level, throwing "PALLEX_USERNAME and PALLEX_PASSWORD are required when PALLEX_MOCK=false"). PALLEX_BASE_URL stays required (mock still intercepts a real URL).

On parse failure, throw `new Error(`Invalid environment: ${issues}`)` where issues lists each failing variable name. Export the parsed object as `export const env = parsed`.

| Boundary | Description |
|----------|-------------|
| process.env → application | Secrets (Pall-Ex username/password, Supabase service role, Upstash token) cross into the app at startup |
| repo → git remote | Source code and config may leak secrets if committed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | .env files committed to git | mitigate | Task 1 adds `.env` and `.env.local` to `.gitignore`; only `.env.example` with empty placeholders is committed |
| T-01-02 | Information Disclosure | Hardcoded credentials in source | mitigate | Task 2 reads all secrets via zod-validated `env` from `process.env` only; no literals in code |
| T-01-03 | Tampering | Misconfigured mock/live switch ships mock to prod or live creds to dev | mitigate | Task 2 makes PALLEX_MOCK an explicit enum-validated boolean; when false, credentials are required (refine), surfacing misconfiguration at startup |
</threat_model>
