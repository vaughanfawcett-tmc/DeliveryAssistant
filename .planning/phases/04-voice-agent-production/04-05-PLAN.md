<objective>
Build the driver-escalation state machine (DRIV-01..04) and the signed `contact_driver`
tool that drives it. When the API can't give a live ETA, offer to contact the driver;
on consent, resolve the driver from the managed list (`getDriverById`), place an outbound
call via the injected `VoiceTelephonyAdapter`, ask for the ETA, relay it back — all under
HARD duration/retry limits (`DRIVER_CALL_MAX_DURATION_S` / `DRIVER_CALL_MAX_RETRIES`),
logging EVERY attempt as a driver/outbound `calls` row linked to the parent customer call.
Unreachable-within-limits yields a callback offer. Fully tested with the MockTelephonyAdapter.

Purpose: Outbound calling is the highest-risk surface (toll fraud, runaway loops). A pure
machine with injected adapter + repo proves the limits and logging offline, with no real call.
Output: escalation machine + contact_driver route + tests.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-19 | Spoofing | contact_driver handler | mitigate | HMAC verify FIRST -> 401; no escalation on unsigned request (test-enforced) |
| T-04-20 | Elevation/SSRF/abuse | outbound destination | mitigate | Destination is NEVER caller-supplied; resolved server-side via getDriverById from the managed list — an unknown/absent driverId yields outcome 'failed', no call placed |
| T-04-21 | Denial of service / toll fraud | outbound call loop | mitigate | Hard maxRetries (default 2 -> max 3 attempts) + maxDurationS (default 180) enforced by the machine; endCall invoked on overrun; no unbounded loop |
| T-04-22 | Repudiation | every attempt | mitigate | Each attempt writes a parent-linked driver/outbound calls row with outcome + duration (DRIV-04) — full audit trail |
| T-04-23 | Elevation | unconsented calling | mitigate | consented:false returns offer-only with zero placeOutboundCall — a call is placed only after explicit customer consent (DRIV-01) |
</threat_model>
