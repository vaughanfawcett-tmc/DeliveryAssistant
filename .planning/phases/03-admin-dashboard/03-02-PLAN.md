<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| database → server (repo) | Raw CallRow.from_number (PII) read from DB; must be masked before crossing toward client |
| seed script → database | Dev script writes sample data; must never run against production data |
| server action → drivers-repo | Mutations (insert/update/delete) reachable from publicly-addressable actions in Plan 03 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Info Disclosure | from_number (caller PII) reaching client | mitigate | `maskPhone` applied inside `listCustomerCalls`; `CallSummary` type has NO raw `from_number` field — masking enforced at the repository boundary and by the type system (Pitfall 4) |
| T-03-08 | Tampering / Info Disclosure | seed script run against production DB | mitigate | Script aborts unless `PALLEX_MOCK==='true'`; `SEED-` tracking_ref prefix makes all sample rows trivially identifiable and removable |
| T-03-09 | Repudiation | silent driver data loss on save failure | mitigate | drivers-repo mutations throw on error (not warn-and-swallow) so the caller surfaces failure to the user |
| T-03-10 | Info Disclosure | service-role client leaking to client bundle | mitigate | Repos access supabase only via lazy dynamic import of the `server-only`-guarded `../supabase` singleton; never imported by client components |
</threat_model>
